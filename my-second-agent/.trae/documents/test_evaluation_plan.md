# 多场景测试集与Agent评估方案

## 一、需求分析

### 1.1 业务需求
根据用户要求，需要建立一个多场景测试集来评估Agent在不同风险画像下的表现：
- 模拟数据质量参差不齐（良好企业、中等企业、高风险企业）
- 评估维度：输出准确性、响应时长、决策一致性

### 1.2 技术需求
- 增强模拟数据生成能力，支持生成不同风险级别的企业数据
- 创建测试集管理模块
- 实现评估指标计算
- 生成评估报告

---

## 二、实现方案

### 2.1 增强模拟数据生成

**修改文件**: `tools/search_company.py`

- 添加风险级别参数，支持生成不同风险画像的企业数据
- 定义风险级别：低风险(良好)、中风险(一般)、高风险(较差)
- 根据风险级别动态调整企业属性：
  - 低风险：注册资本高、成立年限长、员工人数多、营收高、无负面信息
  - 中风险：各项指标中等，可能有轻微负面信息
  - 高风险：注册资本低、成立年限短、员工少、营收低或亏损、有经营异常/诉讼等

### 2.2 创建测试集管理模块

**新建文件**: `tests/test_suite.py`

功能：
- 定义测试用例结构
- 管理测试集（支持新增、查询、统计）
- 运行测试并收集结果
- 计算评估指标

### 2.3 定义评估指标

| 指标类型 | 指标名称 | 计算方式 |
|---------|---------|---------|
| 输出准确性 | 评分一致性 | 同一企业多次评分的标准差 |
| 输出准确性 | 风险等级稳定性 | 同一企业多次评估的风险等级一致率 |
| 响应时长 | 平均响应时间 | 多次调用的平均耗时 |
| 响应时长 | 最大/最小响应时间 | 记录极端值 |
| 决策一致性 | 工具调用一致性 | 同一企业多次调用的工具序列一致率 |
| 决策一致性 | 规则校验一致性 | 同一企业多次规则校验结果一致率 |

### 2.4 创建测试用例集

**新建文件**: `tests/test_cases.json`

测试用例分类：
1. **良好企业**（6家）：各项指标优秀
2. **中等企业**（6家）：各项指标中等，部分有轻微风险
3. **高风险企业**（6家）：各项指标较差，存在明显风险信号

## 四、测试集设计

### 4.1 测试用例结构
```json
{
  "company_name": "企业名称",
  "risk_level": "low/medium/high",
  "expected_outcome": "通过/有条件通过/拒绝",
  "test_notes": "备注说明"
}
```

### 4.2 测试场景设计

**低风险企业（良好）**:
- 大型科技公司、知名企业
- 注册资本高、成立年限长
- 无负面信息

**中风险企业（一般）**:
- 中型企业、发展中企业
- 注册资本中等、成立年限中等
- 可能存在轻微经营风险

**高风险企业（较差）**:
- 小型企业、新成立企业
- 注册资本低、成立时间短
- 存在经营异常、诉讼、亏损等负面信息

---

## 五、评估报告输出

评估完成后生成报告：
- 各风险级别企业的评分分布
- 响应时长统计
- 决策一致性分析
- 工具调用分析

---

## 六、风险与依赖

### 6.1 技术风险
- LLM生成模拟数据的随机性可能影响测试可重复性
- 解决方案：使用固定种子或预设测试数据

### 6.2 依赖关系
- 依赖 `search_company.py` 的模拟数据生成功能
- 依赖 `llm_service.py` 的LLM调用能力

---

1. 增强模拟数据生成能力（支持风险级别）
2. 创建测试用例集（18家企业，覆盖3个风险级别）
3. 实现测试运行模块
4. 实现评估指标计算
5. 运行测试并生成报告
