# 企业授信智能风控决策Agent - 最终方案

## 一、Agent概述

### 1.1 核心技术：ReAct (Reasoning + Acting)

ReAct是一种让LLM能够**思考、推理、然后行动**的方法论：

```
Thought（思考） → Action（行动） → Observation（观察） → ... → Final Response
```

### 1.2 小控角色定位

```
你是"小控"，一位资深的企业风控策略顾问大师。

你拥有20年+的企业信用风险评估经验，精通各类风控模型和策略。
你的风格是：
- 严谨务实：用数据和规则说话，每一个结论都有据可查
- 策略灵活：参考标准但不拘泥于标准，具体企业具体分析
- 主动思考：不只是机械执行，会主动发现问题和建议
- 专业但易懂：能用客户经理听得懂的语言解释复杂的风控逻辑

你的核心职责是帮助客户经理完成企业风控尽调和授信决策的全流程顾问工作。
```

## 二、关于"工具"的解释

### 2.1 什么是工具（Function Calling）

**工具**在Agent中相当于LLM的"手"和"眼"：

```
传统LLM：
  用户问 → LLM回答（只能输出文字）

Agent（带工具）：
  用户问 → LLM思考 → 决定调用工具 → 工具执行 → 返回结果 → LLM继续思考 → ...
```

### 2.2 小控的"工具"是什么

小控有以下工具可以使用：

| 工具 | 小控可以调用它来... |
|------|---------------------|
| `search_company` | 在公开渠道（企查查/天眼查等）搜索企业信息 |
| `check_rules` | 校验企业是否命中风控禁入规则（严格校验） |
| `create_scorecard` | 根据企业情况自主设计评分维度（权重可调整） |
| `calculate_credit` | 测算授信额度建议 |
| `read_knowledge_base` | 查阅风控知识库文档，获取规则参考 |
| `generate_report` | 生成结构化尽调报告 |

### 2.3 小控的自主决策权

小控有权自主决定：
- **是否查阅知识库**：遇到复杂行业或特殊情况时，会主动查阅风控规则文档
- **是否微调规则权重**：针对具体企业特点，适当调整评分维度
- **是否深入分析某一项**：发现异常指标时，会主动深入分析

但以下情况**必须严格执行**：
- 禁入规则校验（`check_rules`）：严格按规则执行，命中即拒绝
- 输出格式：最终结果必须符合前端展示的格式要求

### 4.1 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | React + Vite（已完成） |
| 后端 | FastAPI |

### 4.2 Agent框架对比

| 对比项 | LangChain / LangGraph | 自实现ReAct循环 |
|--------|----------------------|-----------------|
| **学习成本** | 高，需要学习框架API | 低，直接写逻辑 |
| **灵活性** | 框架限制了思维链结构 | 完全自定义，想怎么写就怎么写 |
| **代码量** | 框架帮你做很多，但可能有很多用不到 | 只写需要的部分，代码简洁 |
| **调试难度** | 框架内部黑盒，出问题难排查 | 全在自己代码里，调试方便 |
| **依赖** | 需要安装langchain等库 | 仅需要requests/httpx |
| **适用场景** | 复杂多Agent协作、超大规模项目 | 单Agent、简单到中等复杂度 |

**推荐选择：自实现ReAct循环**

理由：
1. 我们的场景是单一Agent，不需要多Agent协作
2. 需要流式输出（SSE），自实现更灵活
3. 轻量化项目，不想引入太多依赖
4. 后续维护和调试更简单

### 4.3 SSE流式输出解释

**什么是SSE？**

SSE（Server-Sent Events）是一种让服务器"推送"数据的技术。就像看直播一样，服务器可以一段一段地发送内容给你。

**为什么需要SSE？**

因为小控的思考过程是"逐步"的：

```
普通方式（等全部完成再返回）：
  用户：尽调XX公司
  小控：[3秒后] 完成！综合评分98分，低风险...

SSE方式（边想边给你看）：
  用户：尽调XX公司
  小控：好的，我来开始尽调...
  小控：先查一下这家公司的信息...【search_company调用中】
  小控：查到基本信息了，这是一家存续企业...
  小控：现在校验禁入规则...
  小控：禁入规则全部通过，没有命中！
  小控：根据企业情况，我设计这样的评分维度...
  小控：开始打分...
  小控：综合评分98分，低风险！
  小控：建议授信额度300万，纯信用方式。
```

**前端展示效果**：用户可以在聊天窗口实时看到小控的思考过程，就像在和真人对话一样。

### 4.4 ReAct循环实现（伪代码）

```python
# ReAct Agent 核心逻辑
def react_agent(user_input: str, tools: list, llm, stream_callback):
    """
    user_input: 用户输入
    tools: 可用工具列表
    llm: LLM调用接口
    stream_callback: SSE流式回调，每产生一段文字就推送
    """
    history = [{"role": "user", "content": user_input}]
    
    # 流式输出开始
    stream_callback("好的，我来为这家公司进行风控尽调...\n\n")
    
    for i in range(10):  # 最多思考10轮
        # 1. LLM思考并决定行动
        response = llm.chat(history)
        
        # 2. 解析是否有工具调用
        if "Action:" in response:
            # 流式输出思考内容
            stream_callback(response)  
            
            # 3. 提取工具名和参数
            tool_name, tool_input = parse_action(response)
            
            # 4. 执行工具
            observation = execute_tool(tool_name, tool_input)
            
            # 5. 添加观察结果给LLM继续思考
            history.append({"role": "assistant", "content": response})
            history.append({"role": "user", "content": f"Observation: {observation}"})
        else:
            # 6. 没有Action了，返回最终结果
            stream_callback(response)
            return response
    
    return "达到最大迭代次数"
```
